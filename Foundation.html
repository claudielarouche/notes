You can download Foundation here: http://foundation.zurb.com/

When working with foundation, you should do it with SASS

When you first download Foundation, in the CSS folder, there are three files:
- foundation.css: Foundation file
- foundation.min.css: Minified foundation.css
- normalize.css: Normalize.css is a fairly standard style sheet that you can go online and download. And it normalizes all of the defaults that come with browsers

Difference between CSS Reset and Normalize.css

1. Normalize.css preserves useful defaults rather than "unstyling" everything. For example, elements like sup or sub "just work" after including normalize.css (and are actually made more robust) whereas they are visually indistinguishable from normal text after including reset.css. So, normalize.css does not impose a visual starting point (homogeny) upon you. This may not be to everyone's taste. The best thing to do is experiment with both and see which gels with your preferences.

2. Normalize.css corrects some common bugs that are out of scope for reset.css. It has a wider scope than reset.css, and also provides bug fixes for common problems like: display settings for HTML5 elements, the lack of font inheritance by form elements, correcting font-size rendering for pre, SVG overflow in IE9, and the button styling bug in iOS.

3. Normalize.css doesn't clutter your dev tools. A common irritation when using reset.css is the large inheritance chain that is displayed in browser CSS debugging tools. This is not such an issue with normalize.css because of the targeted stylings.

4. Normalize.css is more modular. The project is broken down into relatively independent sections, making it easy for you to potentially remove sections (like the form normalizations) if you know they will never be needed by your website.

5. Normalize.css has better documentation. The normalize.css code is documented inline as well as more comprehensively in the GitHub Wiki. This means you can find out what each line of code is doing, why it was included, what the differences are between browsers, and more easily run your own tests. The project aims to help educate people on how browsers render elements by default, and make it easier for them to be involved in submitting improvements.

Then you have the js folder which contains:

- foundation.min.js: all the js files that are in the foundation folder, minified
- foundation folder: all the js files individually, not minified
- vendor: other js files from other vendors
    - modernizr: useful for cross-browser compatibility, also used for mobile device detection scripts
You can either run the js files individually or you can link to the minified version

This is the barebone of a Foundation html file:

<!doctype html>
<html class="no-js" lang="en">
    <head>
      <meta charset="utf-8" />

      <!-- This line identifies that this will be a responsive web design -->
      <meta name="viewport" content="width=device-width, initial-scale=1.0" />
      <title>Foundation | Welcome</title>

      <!-- You can also change this line to: -->
      <link rel="stylesheet" href="css/foundation.css" />

      <!-- Modernizr can impact the look of the webpage, that's why it is in the head and not at the end of the body -->
      <script src="js/vendor/modernizr.js"></script>
    </head>
    <body>

      <script src="js/vendor/jquery.js"></script>
      <script src="js/foundation.min.js"></script>

      <!--This telling Foundation to monitor any calls to javascript -->
      <script>
      $(document).foundation();
      </script>
    </body>
</html>


The grid

http://foundation.zurb.com/docs/components/grid.html

Foundation ships with a 12-grid system

The first thing to do is to create a div with a class of row:
<div class="row">
</div>

Every row has 12 columns, so you can create as many instances as you want. Next you create your columns:

This is the small grid:
<div class="row">
    <div class="small-3 columns">
    </div>
</div>

We don't have to use only divs. We can also use aside, section, header, footer, nav, etc.

When the screen gets small, the columns will stack on top of each other in a single column. The point at which the stacking occurs happens at a wider point with the large grid and a narrower point with a medium grid.

So to have a different look on a phone, a table and a computer, simply combine the large, medium and small classes within a single div. For example:
<div class="small-9 medium-8 large-2 columns">

Combining all three grids is a little bit overkill. Typically, you'll probably combine the small grid plus either the medium or the large grid.

Offset

Offset is used to introduce more space between columns. Offset is applied on the LEFT of the colum. Example:
<div class="small-9 small-offset-1"> (again you can have an offset in small, medium or large). Make sure your column + offset still equals 12

Offset can even be used on single rows of 12 columns. Example:
<div class="large-10 large-offset-2"> (but it looks kind of stupid to have a large space on the left, oh well, maybe there are some use to it..

Centering

The key here, is to remember that the centering is going to refer to the containing element to which the class is applied. Example:
<div class="large-6 large-centered columns"> Doing that will center the colum on the page but the text itself is not going to be centered

Note:
The order in which the classes are placed don't matter

Block grid

Example: <ul class="medium-block-grid-3">
Which means: Put three pieces of content per row

Ordering

Sometimes on the big screen, you have something in the left column that you actually want to have appear SECOND on the small screen

Foundation offers this way you can change the source order within a row, via push and pull classes. The push classes will push the content from left to right, a given number of cells. The pull classes, pull the content from right to left, a given number of cells.

First step is to rearrange your code so that the SMALL appears properly (so put the code in the right order that you want it to appear on the phone)

<div class="large-3 large-push-9 columns">
    ...
</div>
<div class="large-9 columns large-pull-3">
    ...
</div>

Visibility classes: http://foundation.zurb.com/docs/components/visibility.html

Visibility classes are really handy things. You can show or hide elements of your web size based on your screen size and orientation, or whether the device's touch enabled. But even if an item is not visible, it is STILL loaded to the page. If you want to make some images or content visible at one screen size but not another, there's a better way to do this with Foundation. That's with the Interchange JavaScript component.

Note: Foundation is based in ems (as opposed to Bootstrap which is based in pixels)

      <!-- Remove the class "menu-icon" to get rid of menu icon. Take out "Menu" to just have icon alone -->
      <li class="toggle-topbar menu-icon"><a href="#"><span>Menu</span></a></li> <!--Without the span, the hamburger wasn't showing up for me -->
    </ul>

    <section class="top-bar-section">
      <ul class="right"> <!--If you remove the class of right, the menu will start on the left -->

        <li><a href="#">Meetings & Events</a></li>

</nav>

Equalizer: http://foundation.zurb.com/docs/components/equalizer.html

Makes columns same height (for example, a sidebar and the main content arriving at the same location at the bottom)

1. Add this line to the top: <script src="js/vendor/modernizr.js"></script>
2. Add data-equalizer to the row: <div class="row" data-equalizer>
3. Add data-equalizer-watch to all the sections you want to equalize:
    <article class="medium-9 medium-push-3 columns" data-equalizer-watch>
    <aside class="medium-3 medium-pull-9 columns panel" data-equalizer-watch>

Make image responsive

It's important to replace big images by smaller version for the mobile websites, otherwise people will use up lots of their download for a tiny little image.

Solution 1:
You could use some kind of server side solution. If you happen to be coding with something like PHP, for example, you could have the browser send the client's screen size to the server, and then PHP would pick the right sized image to show up for that particular screen size.

Solution 2:
Javascript based solution by Foundation. The only size downloaded will be the necessary size. Also supports retina images.

Starting point:
<div class="row">
    <div class="medium-12 columns">
      <img src="img/sfnight.jpg" alt="San Francisco." class="bottomspace">
    </div>
</div>

Ending point:
<div class="row">
    <div class="medium-12 columns">
      <img data-interchange="
      [img/interchange/small.jpg, (default)],
      [img/interchange/small.jpg, (small)],
      [img/interchange/retina.jpg], (retina)],
      [img/interchange/medium.jpg, (medium)],
      [img/interchange/large.jpg, (large)]" class="bottomspace"> <!--class of bottomspace is not necessary, it's just needed for this particular project-->
      <!--Adding noscript allow people who have deactivated javascript to still have an image -->
      <noscript><img src="img/sfnight.jpg" alt="San Francisco." class="bottomspace"></noscript>
    </div>
</div>
