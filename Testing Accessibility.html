Testing Accessibility

[camelCaseWordsEnabled: false]

Install Firefox and following toolbars:

- WAVE toolbar
- web developer toolbar
- accessibility evaluation toolbar

1. Is the title related to the page content?

2. Is the content still readable and functional when resized to 200%?
    View > Zoom > Zoom text only
3. Can you navigate the page without the mouse?
    Make sure there are no traps
    Proper tab order
4. Do all images have text version?
    Avoid use of image of text.
    Same image should not necessarily have same alt text
    Firefox extension: Images->Display alt Attributes
    If you have a complicated chart, you can add a text version link underneath which will open the data table or explain in words.
    Decorative images must have empty alt tags
5. Is the page well structured with proper headings?
    heading should not be used to format the text
    WAVE toolbar: Outline
    Toolbar: CSS->Disable Style->Disable all style (make sure everything in proper order because screen reader will read the page in the order it's coded)
6. Where is the link taking you?
    Click every single link to make sure it works
    You don't want to have two link texts going to same place
    Not just "read more", "read more about session 1"
7. Can you see, hear, read and control the video?

8. Is the form clear and easy to use?
    Also make sure there is error validation
    Make sure there is a label for each form control
    Toolbar: Form -> Display Form Details
    Toolbar: Errors, Features and Alerts which will tell you if labels are missing (and other potential problems I suppose)
9. Is the colour contrast sufficient?
    Never use colour to give information ("click on the blue button")
    Colour Contrast Analyzer
10. Can you identify the language in which your content is presented?
    Toolbar: Navigation > Language changes
11. Is the page coded according to specifications?
