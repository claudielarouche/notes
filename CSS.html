Different types of selectors

- Element selectors: Global selectors based on individual HTML elements (example: p, ul, h1, etc.)

- Class selectors: Attribute selector based on class attributes applied to HTML elements
    HTML: <h2 class="subheading"> Item 2 </h2>
    CSS: .subheading {color: blue;}

- ID selectors: Attribute selector based on ID attributes applied to HTML elements (IDs must be unique)
    HTML: <div id="sidebar">[...]</div>
    CSS: #sidebar {font-size: 80%;}
- Element-specific selectors: Modified class or ID selector which limits application to a specific elements. Require more processing work from browser.
    CSS: h2.subheading {color:blue;} (h2s with a class of subheading)
    CSS: div#sidebar {font-size: 80%;} (divs within an ID of sidebar)
- Descendent selectors: Highly specific selectors that target elements based on their location within other elements
    CSS: div p span {color: blue;} (parent selector first followed by each successive nested selector)
      In this example, it would affect span elements that are inside p which are inside a div. More than 3 layers starts to become inefficient.
- Grouping selectors: Allows you to group selectors together that share formatting
    CSS: h1, h2, .quote {font-weight: normal; color: blue;}
- Universal selector: Affects every single element on the page. Author said he never ever needed to use it.
  • {color:blue;}
- Child selectors: Similar to descendant selectors, however they don't apply to every single descendant elements. Only those that are direct descendants.
    article > h2 {color=#7D6855;}
- Adjacent selector: Let you style an element based on which element comes before it in the document, provided that both are inside the same parent. For example you might want to italicize first paragraph right after a heading.
    h2 + p {font-style: italic;}
- Attribute selector (support is not always good in older browsers)
    HTML:
    <p class="red blue green">Red, blue and green classes are applied</p>
    <p class="red">Red is the only one applied</p>

    CSS:
    a[title] {
      color:red; //targets the "a" elements that have a title attribute applied to it
    }

    a[title="Visit Lynda.com"]{
      color:red; //targets the "a" elements that have a SPECIFC title attribute
    }

    p[class="red"]{
      color:red; //will only target the second "p" in the HTML code above (not the first). Exactly the same as saying p.red
    }

    p[class~="red"]{
      color:red; //Added an extra ~ after class. Now it will target first and second "p" from HTML above
    }

    a[href^="http://"] {
      color:red; //Because of ^, it will target everything that has an href attribute that STARTS WITH "http://"
        //It's a good way of targeting external links
    }

    a[href$="pdf"] {
      color:red; //Because of $, it will target everything that has an href attribute that ENDS WITH "pdf"
        //It's a good way of adding a PDF icon next to links that are PDF files
    }

    a[title*="Visit"]{
      color:red; //Because of $, it will target everything that has a title attribute that CONTAINS "Visit"
    }
Pseudo-class selectors

Helps you target elements outside of the DOM (Document Object Model) such as user hovering over an element or focusing on a form element, or elements that are too specific for simple selectors to target.
    :hover -> pseudo-class selector
    a:hover -> targeted pseudo-class selector
    ul a:hover -> descendant selector with a pseudo-class selector
- Dynamic pseudo-class selectors: Target elements based on something other than attributes, content or element type. Usually refers to something that can change over time, or that is based on user interaction.
    a:link {...} //will target but not
    a:visited{color:black;}
    a:hover{background:orange;}
    a:active{background:green;} //Either in focus or currently being clicked on
    //You must put the above selectors in that order, otherwise some styling will overwrite others.
    a:target{color:red;} //it goes with     . If your URL finishes with .html#section1, then the styling target applies to section1
UI Element state pseudo-class selectors: Target specific user interface elements in regards to whether or not they are currently enabled. (relatively new as of 2011)

- Structural pseudo-class selectors: Target elements based on specialized information within the DOM that simple selectors cannot target. This can be pattern-matching, child-to-parent relationship or other structural information.
    span:first-child{font-weight:bold} //Targets span elements that are the first child of their parent. There is also a last-child option.
    h2:first-of-type{border-bottom: 1px solid #42403E;} //Only the first h2 under a parent. There is also a last-of-type option
    p:only-child{font-style:italic;}

    nth:
    - nth-child
    - nth-of-type
    - nth-last-child
    - nth-last-of-type

    Examples of simple nth-child:
    li:nth-child(odd){color:red;} //change color of odd elements (just like iTunes)
    li:nth-child(2) {color:tan;} //change color of second element only
    li:nth-child(2n) {color:tan;} //targets every second element (synonym to saying nth-child(even)
    li:nth-child(2n+1) {color:tan;} //the offset (+1) changes where you start counting. "start to count at 1, and then target every 2nd element"
    li:nth-child(5n+5) {color:tan;} //start to count at 5, and then target every fifth element
    li:nth-child(-2n+1) {color:tan;} //start to count at 8 and then go backward and target every second element (going UP the list instead of DOWN)
    li:nth-child(5n-1) {color:tan;} //start to count at -1 (which does not exist), so the first colored element is 4.

    Examples of complex nth-child:
    li:nth-child(1n+6):nth-last-child(1n+6){color:tan;}
    li:nth-child(-1n+6), li:nth-last-child(-1n+5){color:tan;}
Using Structural pseudo-class selectors helps you to select specific elements without going through classes. Not all supported evenly by all browsers.

Other pseudo-class selectors: Can't be really grouped into any one category. These selectors give you even more specific targeting capabilities based on things like language or URL targeting.

Difference between pixels and ems
Pixels are FIXED units, displayed at the requested size regardless of device or context
Ems are RELATIVE units, displayed relative to the environment in which they are found. 1em = 100% of it's default font size

Sizing with ems

Example 1:

body{
    font-size: 100%;
}

which means the base font size will be: 16px

16px = 1.00em = 100% = 12pt = p
18px = 1.125em = 112.5% = 14pt = h3
24px = 1.50em = 150% = 18pt = h2
48px = 2.250em = 300% = 36pt = h1

Example 2:

body {
    font-size: 62.5%;
}

which means the base font size will be: 10px

So it's easier to convert ems into pixels in your mind thanks to the 10 multiplier.

16px = 1.6em
18px = 1.8em
24px = 2.4em
48px = 4.8em
etc.

Box model http://guistuff.com/css/css_boxmodel.html

Definition: refers to the physical properties of an element's rectangular box. Margins, borders, padding and content width and height.

Properties:

And you can control each of the 4 sizes individually (this is true too for padding and margins)

Total element width = left border + left padding + content width + right padding + right border
    Margin values are not calculated as part of an element's total width
For example:

    padding: 20px;
    border: 1px solid black;
    width: 200px;
}

total width = 1px + 20px + 200px + 20px + 1px = 242px

Considerations:
- If a property is not declared, don't assume that the value is 0
- Elements often have default margins that you need to account for
- A 100% width property when combined with padding and borders will create elements that are larger than their parent

padding: 10px 20px 15px 10px; (applied to top, right, bottom, left)
padding: 10px 20px 15px; (applied to top, right & left, bottom)
padding: 10px 20px; (applied to top & bottom, right & left)
padding: 10px; (applied to all sides equally)

Unlike padding, most elements have a default margin that you must account for

Two elements side by side: Both of the element's margins will apply
Two elements on top of each other: Only one margin will apply (higher value wins)

Positioning elements

3 position schemes:
- Normal flow (static): that's what happens when you do nothing at all
- Element floating
- Absolute positioning

position value:
- static (this is the normal flow)
- relative (still considered part of normal flow but you can offset the element from it's original position relative to another element)
- absolute (not normal flow. Elements are placed relative to the nearest positioned ancester)
- fixed (considered "absolutely" positioned but they are always positioned relative to the active viewport. Meaning if you scroll down, the element stays exactly where it is)
- inherit

As soon as you declare a position that is NOT static, you escape from the normal flow and open up the ability to call on four new properties: top, right, bottom and left

Example relative:
.box1 {
    position: relative;
    left: 100px;
    top: 50px;
}

The box will move 100px to the right and down 50px. It will create a hole where the element would normally be (instead of the other elements to take it's place)

Example absolute:
.box1 {
    position: absolute;
    left: 100px;
    top: 50px;
}

The box will move 100px to the right and down 50px. The other elements under will replace the empty spot and move up.

.content {
    position:relative; //this is done so that the content is now positioned and .quote will be placed based on the top and left of the content. Otherwise, it would use the body of the page as a base.
}

. quote{
    position: absolute;
    left: 20px;
    top: 40px;
}

Example fixed:
.ad {
    position: fixed;
    top: 50px;
    right: 100px;
}

Elements can stack one on top of each other. Can be controlled by the z-index property, which allows you to control which elements are on top of others. By default, positioned elements will appear on top of non-positioned elements if those two overlap.

The default value for z-index is 0. If you want something to go behind your main working layer, give it a negative value. If you want it in front, give it a number greater than 0.

Floats

One of the most popular CSS technique

Definition: Basically an element that's been shifted to the right or to the left of its current position.

.box1 {float: right;}

box1 will move as far to the right as possible. The elements below will move up to occupy the space where box1 was (so floating an element removes it from normal document flow). Different elements react differently to floated elements. "Inline" elements will react to floated elements by wrapping around them if they have enough room.

.content {float:right;}
.sidebar {float: left;}

If you leave just these two, the footer will move up. But if you add this:

.footer{clear: both;}

...then this means that the floating has been cleared. It tells the browser to stop floating elements and go back to normal flow.

Additional learning to develop:
- Learn how floated elements affect their parents
- Learn how clearing works
- Learn how margins work between floated elements and static elements
- Learn how the width of parent elements affect the floating ability of nested elements
- Look into best practices that have evolved around floats
- Learn how browser rendering differences affect layout
- Research multiple layout techniques

Conflicts

p{
    font-size: 1em;
    color: white;
    background: tan;
}
p{
    font-size: 1.2em;
    color: black;
    padding: 10px;
}

In this case, the rules applied will be the following:
- font-size: 1.2em; //From the 2nd p
- color: black; //From the 2nd p
- padding: 10px; //From the 2nd p
- background: tan; //From the 1st p

Cascade
The last rule applied WINS

If you have a style in an external CSS file that conflict with a style in the <style> tag in the HTML file, the HTML file wins, unless the link to the external file comes AFTER the style tag.

Inheritance
If you apply a property to a parent element, all the elements inside of it will have that property applied to it as well

Specificity
Usually the culprit when styling errors.

#mainContent p{color:red;}
p{color:blue;}

You would think that because the "blue" p comes last, it wins. But NO! The one with main content has a higher specificity so it wins!

IDs are worth 100
Classes are worth 10
Elemens and selectors are worth 1

If you are trying to apply a style and it does not work, it could be because there is another selector somewhere with a higher degree of specificity. This is one reason why some people avoid using ID selectors.

Low specificity rules are easier to parse and easier to overwrite later on

!important
In the event of a conflict, the !important declaration wins

p{color:red !important;} //this one wins
#mainContent p {color:green;} //even though this one is more specific

Recommended not to use it much, only in last resort.
Author: I have NEVER used it

Advices
- Avoid using local styles whenever possible
- Do not use inline styles except for very specific circumstances
- Don't mix class and ID selectors without having a plan that guides when they are to be used
- Sections of ID selectors can be hard to overwrite later if not accounted for
- If you find yourself writing descendant selectors with three of more selectors, consider revising your strategy
- If you are familiar with your page structure, you should be able to identify global formatting across your site
- Those formatting needs can be written as global styles on parent elements, and inherited by the rest of the site
- This results in fewer rules to write and easier styles to maintain
- Think about how styles relate to one another
- New designers often make the mistake of styling each element individually, as it is encountered
- This can lead to bloated style sheets and hard to maintain styles
- Thinking of styles as related formatting allows you to plan and write organized style sheets
