Migration through the database

==THIS SET OF INSTRUCTIONS IS OLD AND HAS NOT BEEN REVISED==

Normally you would use Export / Import function.

1. Set up empty Wordpress site on new host

2. Copy plugins, themes and uploads to new host
    Seems like just replacing wp-content folder should do it!
3. Export database from current host through phpMyAdmin
    - Go to phpMyAdmin on source
    - If you're not sure you're in the right database, check wp_options table
    - Click on "Databases"
    - Click on the proper database that's running the wordpress site
    - Click on Export
    - Select Custom
    - Make sure all the database tables are selected
    - Make sure it's set to save output to a file
    - Compression: none
    - Format: SQL
    - Check Add drop table etc.
    - Uncheck Add create procedure etc.
    - Click Go
4. Fix database URLs in text editor
    - Open the file created in step 3 using Notepad++
    - Go to your source and target website main page and get the URL from both, you'll have to replace source with target in the SQL script
    - Search and replace http://localhost/wordpress with http://mor10.com/migration
    - Search and replace /wordpress with /migration
5. Import database on new host through phpMyAdmin
    - Go to phpMyAdmin on target
    - If you're not sure you're in the right database, check wp_options table
    - Check all the tables (under Structure)
    - Click on the arrow next to "with selected", Drop
    - Yes
    - Click Import
    - Browse for file
    - Find your modified SQL script
    - Go
Go to wp-admin
You will see a database update required screen
Click Update
And then you should be good to go but click on links to make sure it's going to the right place (check URL too)

You might have to go to settings > Permalinks and then just change something and put it back as it was and click "Save".

Sometimes the file extension being capital or not makes an image appear properly or not

It's good to match # of posts, # of media, # of comments, etc.
