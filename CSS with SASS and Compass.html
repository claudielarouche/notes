Node.js

When you install Node, you'll also get a really cool utility called NPM. Which stands for Node Package Manager. NPM is going to let you install plugins for Node into your system with a single command. Now later on, we're going to use that to install Grunt. NPM is also a way to extend the functionality of Node.js and your projects.

Now with NPM, you can also use something called a package.json file. That file is going to allow you to specify dependencies that your project needs in order to run. So, say for example that you're going to use Sass in your project. Well, you could go to the Sass website, download the installer, go through the installation, find the appropriate files, install a pre-processor and then get to work once you set up some settings in your pre-processor. Now multiply that times all of the different packages you're going to be using for your project. And what if your working with a team? You would have to write some instructions for all the members of your team to let them know which versions of the packages you're using.

What pre-processors and what the settings are for those pre-processors, now that is obviously a nightmare. By having a package.json file you can have a single file that lists all the packages and the versions you're going to be using in your project.

Example of the content of package.json:

{
    "name" : "rouxmeetups",
    "version" : "0.0.1",
    "dependencies" : {
      "grunt" : "~0.4.1",
      "grunt-contrib-watch" : "~0.5.3",
      "grunt-contrib-compass" : "~0.5.0",
      "grunt-contrib-uglify" : "~0.2.2",
      "matchdep" : "~0.1.2"
    }
}

In command line, go to your project folder (sassycompass for me) and type npm install to run package.json and install your projet's dependencies

More information about package.json: https://docs.npmjs.com/files/package.json

Grunt

1. Install Grunt (in cmd, type "npm install -g grunt-cli"
2. Create gruntfile.js in your main folder
3. Example content of gruntfile.js:

module.exports = function(grunt){
    grunt.loadNpmTasks('grunt-contrib-uglify');

    grunt.initConfig({
      uglify: {
        my_target: {
          files: {
            '_/js/script.js': ['_/components/js/*.js']
          }//files
        } //my_target
      } //uglify
    })//initConfig
} //exports

This basically runs all js scripts in _/components/js and will combine them and minify them in _/js/script.js

To call this script, just go to cmd in the proper folder and type grunt uglify

grunt-contrib-watch

This module allows you to monitor files and do something when the files are modified.

We can add another task to our gruntfile:

module.exports = function(grunt){
    grunt.loadNpmTasks('grunt-contrib-uglify');
    grunt.loadNpmTasks('grunt-contrib-watch');

    grunt.initConfig({
      uglify: {
        my_target: {
          files: {
            '_/js/script.js': ['_/components/js/*.js']
          }//files
        } //my_target
      }, //uglify

      watch: {
        files: ['_/components/js/*.js'],
        tasks: ['uglify']
      } //watch
    })//initConfig
} //exports

To execute, cmd + grunt watch. Now the task will be in a "waiting" state and will execute the uglify command as soon as you change one of the js files. Terminate with CTRL + C

Add this line to gruntfile and when you run grunt, you will have a default task running:
grunt.registerTask('default','watch');

And now you just need to run "grunt"

Livereload

Reloads your Chrome every time you make a change:

module.exports = function(grunt){
    grunt.loadNpmTasks('grunt-contrib-uglify');
    grunt.loadNpmTasks('grunt-contrib-watch');

    grunt.initConfig({
      uglify: {
        my_target: {
          files: {
            '_/js/script.js': ['_/components/js/*.js']
          }//files
        } //my_target
      }, //uglify

      watch: {
        options: { livereload: true },
        scripts: {
          files: ['_/components/js/*.js'],
          tasks: ['uglify']
        }, //scripts
        html: {
          files: ['*.html']
        }
      } //watch
    })//initConfig
    grunt.registerTask('default','watch');
} //exports

In order for this to work, you also need this line in your HTML:
<script src="http://localhost:35729/livereload.js"></script>

Auto compiling SASS files

For SASS to work, you need to have Ruby. Install Ruby from http://rubyinstaller.org

1. In cmd, type gem update --system
2. In cmd still, type gem install compass
3. Create a new file in sassycompass called config.rb

Content:

css_dir = '_/css'
sass_dir = '_/components/sass'
javascripts_dir = '_/js'
output_style = :nested (another option would be compressed)

Any question about this: http://compass-style.org/help/documentation/configuration-reference/

4. Update the grunt file to now include compass stuff
(compass doc: http://compass-style.org/)

module.exports = function(grunt){
    grunt.loadNpmTasks('grunt-contrib-uglify');
    grunt.loadNpmTasks('grunt-contrib-watch');
    grunt.loadNpmTasks('grunt-contrib-compass');

    grunt.initConfig({
      uglify: {
        my_target: {
          files: {
            '_/js/script.js': ['_/components/js/*.js']
          }//files
        } //my_target
      }, //uglify
      compass: {
        dev: {
          options: {
            config: 'config.rb'
          } //options
        } //dev
      }, //compass
      watch: {
        options: { livereload: true },
        scripts: {
          files: ['_/components/js/*.js'],
          tasks: ['uglify']
        }, //scripts
        sass: {
          files: ['_/components/sass/*.scss'],
          tasks: ['compass:dev']
        }, //sass
        html: {
          files: ['*.html']
        }
      } //watch
    })//initConfig
    grunt.registerTask('default','watch');
} //exports

Note: you have to have ruby and compass installed in your system path

5. From terminal: gem install susy
Susy doc: http://susy.oddbird.net/

6. Change config.rb to this:

require "susy"
css_dir = '_/css'
sass_dir = '_/components/sass'
javascripts_dir = '_/js'
output_style = :compressed

Styling

1. Now we're adding some styling in _base.scss

For information about Compass selector helpers: http://compass-style.org/reference/compass/helpers/selectors/

For example:

#{headings()}{
    font-family: $font-secondary
    font-weight: normal;
    color: $darkgrey;
}

You can add clearfix easily in Compass, just like that:

#nextmeeting {
    @include clearfix;

    img{
      width: 30%;
      max-width: 130px;
      float: left;
      margin-right: 3%;
    }
}

And no need to add anything in your HTML, voilà!

For border-radius, you normally have to add all sorts of browser support. With compass, just do this:
@include border-radius(5px);

You can often set up default values. For example, you can add this line to _variables.scss

$default_border_radius : 10px;

Susy

Susy is a framework for creating responsive grids in Compass. Susy provides an additional set of mixins that let's you easily control your layout from within SASS files. One thing I love about Susy, is that it helps you create layouts without having to add additional markup into your HTML files. This is one of the problems with a framework like Bootstrap. When you code layouts, you end up with all kinds of unnecessary classes in your HTML.

You can also establish global settings, to configure Susy for your specific needs. Create a $susy variable, and add your settings as a map.

//Susy settings---------------------------

$susy: (
    columns: 12, // The number of columns in your grid
    column-width: 4em,
    gutters: 1em,
    gutter-position: inside
);

@include border-box-sizing;

$small: 30em;
$medium: 47em;
$large: 75em;

Documentation: http://susy.oddbird.net/


The basic Susy layout is composed using two simple mixins:

@include container; // establish a layout context
@include span(); // lay out your elements
For example:

body { @include container(80em); }
nav { @include span(25%); }
If you want to lay your elements out on a grid, you can use the span mixin to calculate column widths:

nav { @include span(3 of 12); }
But you don’t have to do things the Susy way. We give you direct access to the math, so you can use it any way you like:

main {
    float: left;
    width: span(4);
    margin-left: span(2) + gutter();
    margin-right: gutter();
}

@include span-columns(4 omega, 12); //omega means that this is the last column on the right

Basically if you make the other span-column be the omega, it switches the columns around.

Transition

Documentation: http://compass-style.org/reference/compass/css3/transition/

So you can just add this line (in _layout.scss) to make the big hero picture slide in instead of abruptly appearing:
@include single-transition(all, 1s, ease-out);

Use this code on your NAV to transform it easily into an horizontal list:

@include at-breakpoint($small){
    @include horizontal-list;
    }
Debugging your SASS

In config.rb, you could change this line:
output_style = :compressed

Into this:
output_style = :expanded

So now, instead of having your final CSS file in just ONE line, it is actually expanded and it also tells you where in your scss files stuff are coming from.
