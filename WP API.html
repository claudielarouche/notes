WP API

[camelCaseWordsEnabled: false]

What is it?

The WP API is a RESTful API that can be accessed through simple HTTP requests to access the site data in JSON format

The WP API provides uniform standardized output from Wordpress, which can be requested, parsed and processed by any application or application language.

With the WP API, Wordpress sites are no longer restricted to PHP applications, or even the web browser

The WP API makes Wordpress a headless CMS

Terms

A RESTful application is typically one that uses standard verbs like GET, POST, PUT and DELETE to retrieve and send data from and to a remote server.

RESTful applications use resources, so essentially URIs, to interface with external systems such as a remote server, and use the four verbs to perform operations on that server.

API is an acronym for Application Programming Interface, a set of routines, protocols, and tools for building applications and interfaces.

If we combine these two together, we get a RESTful API, an application programming interface that uses resource URIs to perform operations on a server through verbs. Send a URI to the server, get pure data back. In the WP API, that data comes back as JSON, which stands for JavaScript Object Notation.

So, the WP API is a RESTful API for WordPress that returns data objects in the JSON format when provided with a route and an endpoint.

Endpoints
An endpoint is a function available through the APIs, so the verbs I mentioned in the REST definition. An endpoint performs a specific function by taking one or more parameters and returning the resulting data to the client.

Route
A route is the "name" you use to access the available endpoints. In layman's terms, the route is the URL, and the endpoint is the action performed on the URL.

Example:
GET wp/v2/posts/456

GET = Endpoint
wp/v2/posts/456 = Route

Common Routes

==Posts==

GET http://demo.wp-api.org/wp-json/wp/v2/posts will return 10 latest posts + sticky posts unless you specify a different per_page argument.

GET http://demo.wp-api.org/wp-json/wp/v2/posts?per_page=0 will return every single post

To get a single post:
GET http://demo.wp-api.org/wp-json/wp/v2/posts/{id}
GET http://demo.wp-api.org/wp-json/wp/v2/posts/1

GET http://demo.wp-api.org/wp-json/wp/v2/posts?filter[cat]=4 Get all posts in a specific category

==Pages==
GET http://demo.wp-api.org/wp-json/wp/v2/pages will return 10 latest pages

Similar logic to get more pages or a specific page

==Categories==

GET http://demo.wp-api.org/wp-json/wp/v2/terms/category 10 first categories alphabetically

Same logic for more categories or specific categories

Looks like this may be wrong and it might be this one instead:
GET http://demo.wp-api.org/wp-json/wp/v2/categories

==Tags==

GET http://demo.wp-api.org/wp-json/wp/v2/tags

Morten said to use this but it doesn't seem to work:
http://demo.wp-api.org/wp-json/wp/v2/terms/tag

==Users==

http://demo.wp-api.org/wp-json/wp/v2/users
http://demo.wp-api.org/wp-json/wp/v2/users/me Returns your user if you are logged in

==Comments==

http://demo.wp-api.org/wp-json/wp/v2/comments
http://demo.wp-api.org/wp-json/wp/v2/comments?post=49 Returns comments on a single post. You can comma separate those values to return more than one post.
