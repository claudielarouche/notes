Regular Expressions

[camelCaseWordsEnabled: false]

d : used in place of any digit from 0 to 9
D : represents any non-digit character
. : match any single character (in order to specifically match a period, you need to escape the dot by using a slash . accordingly.)
w : used in place of any alphanumeric
W : any non-alphanumeric
s : any white space
S : any non-whitespace character
[abc] : will only match a single a, b, or c letter and nothing else.
[^abc] : will instead match any single character except for the letters a, b, or c.
[0-6] : will match any digit character from zero to six, and nothing else.
[^n-p] : will match any character except for letters n to p.
a{3} : will match the a character exactly three times. Some implementations, also allow you to specify a range for this value. For example, "a{1,3}" will match the a character no more than 3 times, but no less than once. In Javascript, you can even leave out the second count and just specify a minimum value, for example "a{1,}" will match 1 or more a's.
d* : matches any number of digits
d+ : matches one digit or more
? : denotes optionality. Allows you to match either zero or one of the preceding character or group. For example, the pattern ab?c will match either the strings "abc" or "ac" because the b is considered optional.
^ : matches the beginning of the string
$ : matches the end of the string
() : capture a part of the pattern. Example: (w+).pdf$ will capture file_a from file_a.pdf
| : logical OR, aka. the pipe. Denotes different possible sets of characters. I love (cats|dogs)
b : matches the boundary between a word and a non-word character

Patterns are case sensitive and a-z differs from A-Z in terms of the characters it matches (lower vs upper case).

The most common forms of whitespace you will use with regular expressions is defined as the space _, the tab t, the new line n and the carriage return r (useful in Windows environments), and these special characters match each of their respective whitespaces. In addition, a whitespace special character s will match any of the specific whitespaces above and is extremely useful when dealing with imperfect input text.

One concept that we will not explore in great detail in these lessons is referencing, mostly because it varies depending on the implementation. However, many systems allow you to reference your captured groups by using 0 (usually the full matched text), 1 (group 1), 2 (group 2), etc. This is useful for example when you are in a text editor and doing a search and replace using regular expressions to swap two numbers, you can search for "(d+)-(d+)" and replace it with "2-1" put the second captured number first, and the first captured number second.
